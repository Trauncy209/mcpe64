# Contributing to TruancyCraftPE

Thanks for wanting to help improve **Minecraft Pocket Edition 0.6.1 alpha**.

This project welcomes:
- engine and gameplay programmers
- Android and build-system contributors
- UI/UX contributors
- testers on real devices
- bug reporters
- classic MCPE players with feedback

## Project Direction
We are trying to make classic Minecraft PE the best version of itself:
- keep the original identity and feel
- improve compatibility on modern devices
- fix broken or incomplete systems
- add quality-of-life improvements carefully
- prefer changes that make the game feel more polished, playable, and stable

## Before You Start
Please try to:
- keep changes focused and easy to review
- explain what problem you are solving
- avoid unrelated cleanup in the same PR
- test on-device when changing Android behavior
- include screenshots or logs for UI/device-specific fixes when possible

## Good Contributions
Examples of helpful contributions:
- save/load fixes
- screen scaling fixes
- controller/input improvements
- render/performance fixes
- options menu improvements
- multiplayer/LAN fixes
- crash fixes
- documentation and setup improvements

## Pull Request Guidelines
When opening a PR:
1. use a clear title
2. explain the problem
3. explain the fix
4. list how you tested it
5. attach screenshots/logs if relevant

Small, well-tested pull requests are preferred over giant rewrites.

## Code Style
For now, follow the existing style in the area you touch:
- keep naming and formatting consistent with nearby code
- prefer minimal, understandable patches
- avoid large refactors unless they are necessary

## Issues and Feature Requests
If you are not ready to code yet, you can still help by opening issues for:
- bugs
- device compatibility problems
- UI/UX problems
- gameplay regressions
- feature ideas that fit the project mission

## Community
Players and developers are both welcome.

If you love classic MCPE and want to help preserve and improve it, you belong here.
